# CLAUDE.md — RubySapphireRecomp

This game repo defers all rules to the platform core:

- `../gbarecomp/CLAUDE.md`     — execution contract
- `../gbarecomp/PRINCIPLES.md` — recomp principles
- `../gbarecomp/DEBUG.md`      — debug loop
- `../gbarecomp/TCP.md`        — TCP debug protocol (Ruby 19872 / Sapphire 19882)

## Two variants, one engine

Ruby and Sapphire are separate CMake targets (`RubyRecomp`,
`SapphireRecomp`) built from one source tree via `add_gba_variant()` in
`CMakeLists.txt` — the Sonic3AndKnucklesRecomp pattern. Per-variant data
lives under `variants/<name>/`. A fix in `src/` or the engine applies to
both; per-game facts live in each `variants/<name>/game.toml`.

## Build target

Builds against the **live `../gbarecomp` checkout on `main`**. Build with
MSYS2 mingw64 + Ninja, invoked from PowerShell (the Bash sandbox forces
`TEMP=C:\Windows`, which breaks `cc1plus`).

## Game-specific rules

1. **The decomp is a reference, not an oracle.** `pret/pokeruby` is
   imported through `tools/import_decomp_symbols/` (names, boundaries,
   ROM-layout). Its C source / build output / toolchain never enter.
2. **mGBA is the oracle.** mGBA wins over the recomp unless a hardware
   test ROM shows otherwise.
3. **ROM hash gates everything.** Ruby rev1 `610b96a9…`, Sapphire rev1
   `4722efb8…` (pokeruby `*_rev1.sha1`). Versions are added by checksum in
   `variants/<name>/config/<region>.toml`.
4. **No Ruby/Sapphire special cases in the GBA core.** Hardware-corner
   fixes live in `gbarecomp/src/gba/` with a citation.
5. **Generated C is never edited.** `variants/*/generated/*` is
   regenerated by `gba_recompile`. If output is wrong, fix the recompiler.
6. **Save chip:** 1 Mbit flash + RTC (both games). RS is the oldest Gen3
   engine; expect early-boot and flash/RTC differences from FRLG/Emerald.
